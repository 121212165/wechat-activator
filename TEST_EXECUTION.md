## 二、当前已知问题

### 问题1：编辑关键词/公众号后无法保存

| 属性 | 内容 |
|-----|------|
| **问题描述** | 点击编辑关键词，修改后点击保存，提示"更新失败" |
| **影响功能** | 关键词编辑、公众号编辑 |
| **根本原因** | 数据库RLS策略缺少UPDATE权限 |
| **修复方案** | 执行迁移文件 00002_fix_update_policies.sql |
| **状态** | ⏳ 待执行 |

### 问题2：搜狗API调用失败

| 属性 | 内容 |
|-----|------|
| **问题描述** | 点击刷新，搜狗抓取失败，始终返回模拟数据 |
| **影响功能** | 文章抓取（真实数据源） |
| **根本原因** | 搜狗微信搜索页面结构变化，HTML解析正则失效 |
| **修复方案** | 更新parseArticlesFromHTML解析逻辑（多种后备方法） |
| **修复文件** | supabase/functions/fetch_articles_real/index.ts |
| **部署方式** | 需要重新部署Edge Function |
| **状态** | ✅ 已修复（代码层面），待部署 |

### 修复内容

1. 方法1：新版页面结构提取
2. 方法2：article-box备用结构
3. 方法3：通用链接提取
4. 去重逻辑

**部署命令**：
```bash
supabase functions deploy fetch_articles_real
```

**如果仍无法抓取**，可能原因：
- 搜狗有反爬机制（需要代理/IP）
- 页面结构已大幅变化
