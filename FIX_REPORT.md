## 一、问题总结

### 问题1：编辑后关键词无法保存

**问题描述**：
- 用户点击编辑关键词，修改后点击保存，提示"更新失败"
- 公众号编辑也存在同样的问题

**根本原因**：
- 数据库RLS策略缺少UPDATE权限
- keywords表和accounts表只有SELECT、INSERT、DELETE策略
- 没有UPDATE策略，导致用户无法更新自己的数据

**影响范围**：
- 所有用户无法编辑关键词
- 所有用户无法编辑公众号
- 影响用户体验，需要删除后重新添加

---

### 问题2：没有实现API的接入，数据重复和落后

**问题描述**：
- 文章抓取存在重复数据
- 抓取的文章时间过旧
- 没有时间过滤机制

**根本原因**：
- 去重逻辑不完善，只检查标题，不检查链接
- 没有时间过滤，抓取了很久以前的文章
- 去重检查在插入前进行，效率低

**影响范围**：
- 用户看到重复的文章
- 用户看到过时的文章
- 数据库存储浪费

---

## 二、修复方案

### 修复1：添加UPDATE策略

**修复内容**：
```sql
-- 关键词表UPDATE策略
CREATE POLICY "用户可以更新自己的关键词" ON keywords
  FOR UPDATE TO authenticated USING (auth.uid() = user_id);

-- 公众号表UPDATE策略
CREATE POLICY "用户可以更新自己的公众号" ON accounts
  FOR UPDATE TO authenticated USING (auth.uid() = user_id);
```

**修复文件**：
- `supabase/migrations/00002_fix_update_policies.sql`

### 修复2：改进文章抓取和去重逻辑

**修复内容**：

#### 2.1 添加时间过滤
```typescript
// 时间过滤：只获取最近7天的文章
const DAYS_FILTER = 7

// 获取用户最近7天的文章标题（用于去重）
const sevenDaysAgo = new Date(Date.now() - DAYS_FILTER * 24 * 60 * 60 * 1000).toISOString()
```

#### 2.2 改进去重逻辑
```typescript
// 获取用户最近7天的文章标题和链接
const {data: existingArticles} = await supabase
  .from('articles')
  .select('title, link')
  .eq('user_id', user_id)
  .gte('created_at', sevenDaysAgo)

const existingTitles = new Set<string>()
const existingLinks = new Set<string>()
if (existingArticles) {
  existingArticles.forEach((a) => {
    existingTitles.add(a.title)
    if (a.link) existingLinks.add(a.link)
  })
}

// 检查是否重复（标题或链接）
const isDuplicate =
  existingTitles.has(article.title) ||
  (article.link && existingLinks.has(article.link))
```

#### 2.3 时间验证
```typescript
// 检查发布时间是否在7天内
const publishedAt = new Date(article.published_at)
const isRecent = publishedAt >= new Date(sevenDaysAgo)

if (isRecent) {
  // 添加到待插入列表
  articlesToInsert.push({...})
  // 添加到去重集合
  existingTitles.add(article.title)
  if (article.link) existingLinks.add(article.link)
}
```

**修复文件**：
- `supabase/functions/fetch_articles_real/index.ts`

**验证方法**：

2. 配置关键词和公众号
3. 进入首页，点击刷新
4. 等待抓取完成
5. 检查文章列表

**预期结果**：
- ✅ 不显示重复的文章
- ✅ 只显示最近7天的文章
- ✅ 文章按时间排序
